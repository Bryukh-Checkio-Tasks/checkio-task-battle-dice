For this task, you'll be playing some board games. Or rather, you're going to be, but first you'd like to know more about the probabilities involved.

Many board games fall under the category of "roll and move" games. In these games, you roll some dice and move that many spaces forward. In some games, you then move forward or backwards a number of spaces depending on which space you landed on.

You want to work out, given a description of the board and the dice used, what the expected number of moves would be to reach a target square. For this task, all the boards under consideration are circular and when moving off one end you simply continue from the other.

Input: A number of dice to roll, N, the number of sides on each die, S, a target space, T, and a list, B, containing how many spaces to move (positive for forward, negative for backwards) after landing on that space. You always start from space 0 and the starting and ending squares will not require any further movement after landing on them. Any further movement will move you to a space that does not require further futher movement.

Output: The expected number of moves needed to reach the target space starting from space 0, correct to 1 digit after the decimal place. If it takes 33 1/3 moves, you should return 33.3.

Example: 

checkio(2, 6, 3) == 0.0556  # 2 six-sided dice have a 5.56% chance of totalling 3
checkio(2, 6, 4) == 0.0833 
checkio(2, 6, 7) == 0.1667
checkio(2, 3, 5) == 0.2222  # 2 three-sided dice have a 22.22% chance of totalling 5
checkio(2, 3, 7) == 0       # The maximum you can roll on 2 three-sided dice is 6
checkio(3, 6, 7) == 0.0694
checkio(10, 10, 50) == 0.0375

How it is used: Some of the methods for solving this task can be used to tackle a wide variety of other problems.

Preconditions:
1 ≤ N ≤ 3
2 ≤ S ≤ 10
0 < T < len(B)
4 ≤ len(B) ≤ 30
B[0] = B[T] = 0
B[(i + B[i]) % len(B)] = 0 for all i
